Map Lat/Long

Google Earth relies on latitude and longitude information to plot exact locations on a map. This article describes how to map a database table's latitude and longitude values for use in a given connection's Google Earth KML file.

Assumptions/Prerequirements

In order to map latitude and longitude for a connection, you must first:

  1. Create a connection
  2. Edit a connection

Help for determining which latitude and longitude columns to select is included in the Notes section below.

Instructions

  1. Select a table from the list of database tables on the left-hand side of the Connection Tables/Columns module. After selecting a table, you will be provided with a list of its columns on the right-hand side and three options: Map Name, Map Lat/Long, and View Table.
  2. Select Map Lat/Long.

  3. You can map latitude and longitude as two separate columns or together as one column. See pictures in Step 4 for more details.
    To map the columns separately, click the Separate button. To map latitude, select the column with the latitude values from the drop-list. To map longitude, select the column with the longitude values from the other drop-down list.
    To map the columns together, click the Together button. You will only select one column with this option. However, you must select whether the data is included as Latitude/Longitude or Longitude/Latitude by clicking the appropriate radio button. If you are unsure what the correct order is, check out the Notes section below.
  4. One you have selected the correct columns to map, click the Submit button to map the latitude and longitude values for the connection.

  5. The new latitude and longitude mappings will be displayed at the top of the module.

  6. To save the mappings, see Save Connection.

Warnings

  • A placemark name must also be selected in addition to the latitude and longitude columns for the locations to be mapped properly in Google Earth. For a description on how to add a placemark, see Map Placemark Name.
  • By selecting a table to map latitude and longitude values to, you are selecting which table to draw data from for use in Google Earth. Currently, only one table's data can be displayed per KML file. Because of this, any conditions that are created for icons and overlays that involve other database tables will not be taken into account.

Notes

Selecting Appropriate Columns

If you are unsure which columns to select as the appropriate latitude and longitude columns, you can view all of the data in any of the database tables by selecting the View Table button after selecting a database table.

Mapping Lat/Long Together

ODBC2KML handles parsing of columns mapped as both latitude and longitude. Because of this, you don't have to worry with including anything other than what order the coordinates are.
