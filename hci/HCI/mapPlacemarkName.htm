Map Placemark Name

Google Earth relies on a placemark name to give an identifier to each location on a map. This article describes how to map a database table's placemark name for use in a given connection's Google Earth KML file.

Assumptions/Prerequisites

In order to map a placemark name for a connection, you must first:

  1. Create a connection
  2. Edit a connection

Help for determining which placemark name column to select is included in the Notes section below.

Instructions

  1. Select a table from the list of database tables on the left-hand side of the Connection Tables/Columns module. After selecting a table, you will be provided with a list of its columns on the right-hand side and three options: Map Name, Map Lat/Long, and View Table.
  2. Select Map Name.

  3. Select the appropriate column from the drop-down list as your placemark name. One you have selected the correct column to map, click the Submit button to map the placemark name value for the connection.

  4. The new placemark mapping will be displayed at the top of the module.

  5. To save the mappings, see Save Connection.

Warnings

  • Latitude and longitude columns must also be selected in addition to the placemark name for the locations to be mapped properly in Google Earth. For a description on how to add latitude and longitude, see Map Lat/Long.
  • By selecting a table to map a placemark name to, you are selecting which table to draw data from for use in Google Earth. Currently, only one table's data can be displayed per KML file. Because of this, any conditions that are created for icons and overlays that involve other database tables will not be taken into account.

Notes

Selecting Appropriate Columns

If you are unsure which column to select as the appropriate placemark name, you can view all of the data in any of the database tables by selecting the View Table button after selecting a database table.
