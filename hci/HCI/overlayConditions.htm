Overlay Conditions

Conditions can be added to overlays to specify to the KML generator when the given overlay should be used.

Assumptions/Prerequirements

In order to add conditions to an overlay, you must first:

  1. Add an overlay to the connection

Instructions For Adding a Condition to an Overlay

  1. Next to an overlay, click Modify Conditions
  2. Enter a valid condition (see Notes section)
  3. Click Add
  4. Repeat steps 2 and 3 for each condition you want to add to the overlay
  5. When you are finished adding conditions, click Submit

Instructions For Removing a Condition from an Overlay

  1. Next to an overlay, click Modify Conditions
  2. Click Remove next to each condition you want to remove from the overlay
  3. When you are finished removing conditions, click Submit

Warnings

When adding conditions to an overlay, be sure to hit Add before hitting submit, or the condition will not be added.

Notes

Only conditions that are in the same table as your mapping information will actually be used.
When using multiple overlays, only the first overlay that has a positive match on a condition will be put into the KML.
Clicking Cancel disregards any changes made inside the overlay condition popup.
Tips for valid conditions:

  • You must enter a lower bound or an upper bound
  • If you enter a lower bound, you must enter a lower operator
  • If you enter a lower operator, you must enter a lower bound
  • If you enter an upper bound, you must enter an upper operator
  • If you enter an upper operator, you must enter an upper bound
  • If you enter both an upper bound and a lower bound, both must be either numbers or strings; you cannot mix and match
  • If you use the == operator, you can only enter the corresponding bound; you cannot mix the == operator with another operator
  • If you use the != operator, you can only enter the corresponding bound; you cannot mix the != operator with another operator
  • If you use both an upper bound and a lower bound, the lower bound must be less than the upper bound
  • Only numeric bounds can be used with the < or <=; strings cannot be used
