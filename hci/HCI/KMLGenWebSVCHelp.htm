KML Generation through Web Service Help Page

KML Generation is one of the key features of the ODBC2KML project.
This feature allows for the user to dynamically create KML on-the-fly
for a given ODBC2KML connection. This feature is extremely useful for
data that might be routinely updated, or when you need a machine readable
link to the ODBC2KML system.

Assumptions/Prerequisites

KML Generation will not work properly if the following do not all ready exist:

  1. A connection (connection information)
  2. Mapping for that connection

There are many other things that can be set up when you create a connection;
however, those are the only two things needed for KML generation to generate
a complete KML file.

Instructions

  1. Log into ODBC2KML.
  2. Find the connection you would like to create a KML file for on the main page.
  3. Click the Edit Connection button to the right of the bar containing that connection.
  4. Click Save and Edit on the connection information box that pops up.
  5. At the top of the Connection Details page, copy the link to the KMLGenWebSVC.
  6. Paste the link into your application of choice.
For example, to use the web service link in Google Earth:
  1. In Google Earth, click Add and then Network Link.
  2. In the Name box, give the connection a descriptive name.
  3. Paste the link to the web service in the Link box and click OK.
  4. The KML should show up in the main Google Earth window.

Warnings

If connection is lost to the database mapped by the connection, or the connection
is lost to the server hosting the project, the KML file will not generate.

Notes

Important: The KML file can be generated without overlay colors, icons, or a description.
Thus, if you download a KML file with default overlays and icons, do not assume the file is incorrect.
Instead, make sure that the connection has been populated with icon and overlays.
