Save Connection

Before you can use the KML Generation Web Service to generate KML files from a given connection, the connection must be saved.
This article will walk you through saving a connection.

Assumptions/Prerequirements

In order to save a connection, you must first:

  1. Create a connection
  2. Edit a connection
  3. Map a latitude and longitude

You may also setup a description, any icons, and any overlays before saving the connection.

Instructions

  1. Select Save Connection at the bottom of the page
  2. Select Yes on the warning dialog.

Warnings

When you select Yes on the warning dialog the system will sanitize the connection.



Sanitizing the connection does the following:

  • Validates the current connection information
  • Checks the current mapping to ensure that the proper tables and fields exist in the database. If not, the connection will not save
  • Ensures that the description field does not contain any invalid tags. If it does the description will be cleared
  • Any conditions that use table name/ field name combinations that no longer exist in the remote database will be deleted
