KML Generation through Web Application Help Page

KML Generation is one of the key features of the ODBC2KML project.
This feature allows for the user, from the main page, to create and open or
save a KML file relative to any existing connection.

Assumptions/Prerequisites

KML Generation will not work properly if the following do not all ready exist:

  1. A connection (connection information)
  2. Mapping for that connection

There are many other things that can be set up when you create a connection;
however, those are the only two things needed for KML generation to generate
a complete KML file.

Instructions

  1. Log into ODBC2KML.
  2. Find the connection you would like to create a KML file for on the main page.
  3. Find the KML button on the far right of the bar containing that connection.
  4. Click KML.
  5. Save or open the file.

Warnings

If connection is lost to the database mapped by the connection, or the connection
is lost to the server hosting the project, the KML file will not generate.

Notes

Important: The KML file can be generated without overlay colors, icons, or a description.
Thus, if you download a KML file with default overlays and icons, do not assume the file is incorrect.
Instead, make sure that the connection has been populated with icon and overlays.
